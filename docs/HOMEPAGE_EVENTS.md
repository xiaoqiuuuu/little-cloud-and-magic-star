# 官网活动管理说明

官网首页不再绑定某一场线下活动，而是由独立的“官网活动”数据驱动。

## 地址规则

- `/`：展示后台选定的当前主页活动。
- `/events/{slug}`：每场活动的固定地址。
- 已切换下来的活动不会被覆盖，仍可通过固定地址访问。
- 首页不显示活动切换控件；往期活动只通过各自的固定地址访问。

## 举办下一场活动

1. 使用拥有“主页活动管理”权限的账号进入“后台管理 → 官网活动”。
2. 在上一场活动右侧点击“复制”。
3. 修改活动名称、固定网址标识、日期、地点、首页文案和物料。
4. 保存后，该活动仍是草稿，不会被普通访客看到；可以使用“预览”检查实际首页效果。
5. 点击“设为主页”并确认，活动会自动发布，根路径 `/` 会立即切换。
6. 确认旧活动不再需要作为常用活动展示后，可以将它归档；固定链接仍然有效。

固定网址标识只能使用小写字母、数字和短横线，例如：

```
summer-shenzhen-2026
```

对应地址为：

```
/events/summer-shenzhen-2026
```

## 状态含义

- 草稿：只有拥有主页活动管理权限的账号能在后台看到，不能从公开链接访问。
- 已发布：可公开访问，也可以随时设为主页。
- 往期：仍可通过固定地址公开访问，但不能直接归档当前主页活动。

活动第一次发布后，固定网址标识会锁定，避免已经分享出去的链接失效。

## 内容范围

每场活动可以独立配置：

- 活动名称、日期、地点和固定网址；
- 首页主标题、介绍和配色主题；
- 规则入口及其链接；
- 任意数量的物料卡片和图片；
- 获取方式、版权和页脚说明。

数据库升级时会自动把原有“小雲殺 2.0 · 深圳收官站”首页写入首个官网活动，并保持为当前主页，因此部署后原首页内容会继续显示。
